Management

GitHub Dev Site Workflow Summary

This section outlines the process undertaken to establish the current development workflow for the Redsun2020:Revise DEV documentation site, utilizing GitHub, GitHub Pages, and the Jekyll Static Site Generator.

1. Initial GitHub Repository Setup

  • A new repository named Redsun2020-Revise-Dev was created on GitHub (https://github.com/Jon-kurishita/Redsun2020-Revise-Dev).
  • Basic repository settings were configured (e.g., initialized with a README, LICENSE).

2. Local Project Setup

  • The GitHub repository was cloned to the local development machine using the command:
    git clone https://github.com/Jon-kurishita/Redsun2020-Revise-Dev.git
  • The local project folder is located at I:\08 REDSUN2020\Github\Redsun2020-Revise-Dev.

3. Jekyll Migration

  • Recognizing the need for scalability and easier maintenance (especially for navigation), we decided to migrate the site structure from plain static HTML files to the Jekyll Static Site Generator (SSG).
  • A backup of the original static HTML structure was created using a Git branch named backup-static-html.

4. Local Environment Setup (PC)

  • Required tools: Ruby+DevKit, Jekyll, and Bundler.
  • Installation was performed using RubyInstaller for Windows (including DevKit).
  • Jekyll and Bundler were installed globally using the RubyGems command: gem install jekyll bundler.
  • Installation was verified using ruby -v, jekyll -v, and bundle -v in a **new** terminal window.
  • A Gemfile was created in the project root to specify dependencies (using the github-pages gem for compatibility).
  • Project-specific gems were installed using bundle install.

5. Current Jekyll Project Structure

  • _config.yml: Main Jekyll configuration file (includes baseurl: /Redsun2020-Revise-Dev).
  • _layouts/default.html: Contains the main HTML template (boilerplate, header, footer, CSS/JS links) with the {% raw %}{{ content }}{% endraw %} placeholder where individual page content is inserted.
  • _includes/navigation.html: Contains the complete, reusable navigation structure (multi-level, expandable). Included in the layout via {% raw %}{% include navigation.html %}{% endraw %}.
  • assets/: Folder containing subfolders for css and js.
  • Content Files (index.html, 1-project.html, *.html): Located in the root directory. Each contains YAML Front Matter (specifying layout: default and a `title`) followed only by the unique HTML content for that page.

6. Local Development Workflow

  1. Make changes to source files (e.g., edit content in page files like 1-project.html, update navigation in _includes/navigation.html, modify styles in assets/css/styles.css).
  2. Open a terminal in the project root directory (I:\08 REDSUN2020\Github\Redsun2020-Revise-Dev).
  3. Run the command bundle exec jekyll serve to build the site and start the local preview server.
  4. Open the server address (usually http://127.0.0.1:4000/Redsun2020-Revise-Dev/) in a web browser to preview changes.
  5. When finished previewing, stop the server by pressing Ctrl+C in the terminal.

7. Publishing Updates to GitHub Pages

  1. Open a terminal in the project root directory.
  2. Use the combined Git command:
    git add . && git commit -m "Your descriptive commit message" && git push origin main
  3. Wait a few minutes for GitHub Pages to automatically rebuild and deploy the site using Jekyll.
  4. Verify changes on the live site: https://jon-kurishita.github.io/Redsun2020-Revise-Dev/
